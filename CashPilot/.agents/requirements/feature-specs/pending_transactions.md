# Pending Transactions

**Goal**: Staging area for transactions needing review/sync before committing to the main ledger.

## Workflow
1. **Staging**: Transactions created in "Pending" state (cash flows, bank movements, etc.).
2. **Review & Submit**: Users review list, then "Submit" to move pending → permanent ledger.

## Acceptance Criteria
- [ ] View all pending transactions.
- [ ] Delete pending transactions before submission.
- [ ] "Sync" correctly converts pending → permanent.
