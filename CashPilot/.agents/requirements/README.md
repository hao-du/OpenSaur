# Business Requirements

This directory contains the **"What"** of the application (vs. `.agents/rules/` which defines **"How"**).

- **`core-business-logic.md`** — Domain-wide rules and constraints.
- **`feature-specs/`** — Detailed functional specs per feature.

**Usage**: Check before implementing (business logic & edge cases), during implementation (correctness), and document any undocumented rules discovered via code analysis.
